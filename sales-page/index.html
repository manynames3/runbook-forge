From noisy incident evidence to procedural memory.

Runbook Forge makes operational learning concrete. It classifies evidence, writes a reviewable report, records the fingerprint, and proposes a reusable skill only after a pattern repeats.

01

Ingest evidence

Terraform plan JSON, fake AWS fixtures, CI failure metadata, logs, and repo docs.

02

Classify risk

Root-cause classifiers produce severity, blast radius, and recommended checks.

03

Track recurrence

Deterministic fingerprints turn repeated incidents into pattern memory.

04

Propose skills

Runbook drafts capture investigation steps, safe commands, approvals, and rollback.

Built for the failure modes CloudOps teams actually review.

The project is fixture-backed and deterministic, so reviewers can run it locally without cloud credentials or network calls.

TF

Terraform plan review

Detects public S3 exposure, world-open security groups, wildcard IAM, destructive RDS changes, and NAT Gateway cost risk.

EC

ECS deployment triage

Correlates CI output, service events, target health, stopped tasks, and container logs to classify deployment failures.

SQ

SQS/Lambda backlog triage

Flags throttling, timeout pressure, poison messages, batch mismatch, DLQ growth, and downstream dependency errors.

Recommendation-first automation with hard safety boundaries.

Runbook Forge is deliberately not an auto-remediator. It generates proposed commands, report text, and approval checklists, while write actions stay blocked behind humans.

read_only

Safe evidence inspection and fixture parsing.

propose_only

Rollback, scaling, and Terraform commands rendered as text.

requires_human_approval

No infrastructure-changing action runs without explicit approval.
